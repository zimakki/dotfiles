Why the brief is visual first

Why this looks like your terminal

Every tool in this machine's daily stack — bat, ghostty, starship, atuin, lazygit — renders in Catppuccin Mocha. A generated document that arrives styled any other way is a context switch; one that arrives in Mocha is just another pane. So the page is built from the palette's own semantic tokens: base for the page, mantle for code panes, mauve for structure — the same accent your atuin theme uses for titles.

The one deliberate departure from the terminal: prose is set in Charter, a serif that ships with macOS and was designed to stay sturdy at low contrast. Explainers are for reading, and a full page of monospace is a transcript, not a document. Everything structural — headings, labels, code — stays mono, so the two worlds are always distinguishable at a glance.

Typography

Three roles, all from fonts already on the machine — no webfonts, nothing fetched:

  • Headings — SF Mono, semibold, in mauve. Each heading wears its own markdown syntax: the dimmed ## prefix is the hierarchy, in the same notation the agent authored it and your feedback travels in.
  • Body — Charter at 17px with a 1.75 line height, tuned for long-form reading on a dark ground.
  • Utility — small SF Mono for section anchors, code-pane language labels, captions, and the feedback button.
On a wide window, look left of this section's heading: the #typography label in the margin is the exact anchor your copied feedback will cite. The margin isn't decoration — it's the annotation system made visible.

Heading depth is a color, not just a size

Third-level headings render in green — use them to break a section into named steps or facets without promoting them to the contents rail (only data-note sections appear there).

And the fine print is yellow

Fourth-level headings are for edge cases, caveats, and asides that still deserve a scannable label. Deeper nesting than this usually means the section wants to be split.

Layout and measure

Prose holds a measure of about seventy-two characters — a 48rem column at 18px. On a laptop the page is deliberately asymmetric, like a docs site: the contents rail takes the left margin, the text column leans left-of-center, and code panes and diagrams grow rightward from the prose edge — up to 60rem — into the space that would otherwise sit empty. Code is read by shape as much as by line; squeezing it to prose width forces wrapping where it hurts most.

Only on very wide displays does the column drift back toward center. Below about 1344px the rail folds away; below 1216px the margin anchors follow, and the document becomes a plain single column — the 💬 affordances remain, so annotation still works on a half-screen or a phone.

Code, in your own theme

Code panes are drawn the way bat draws them in a ghostty window: mantle ground, hairline border, a dimmed language label in the corner. Syntax colors follow the official Catppuccin mapping — mauve keywords, green strings, blue functions, peach numbers — as ten CSS token classes the generating agent applies by hand. No highlighter library, no runtime cost, nothing to load: 

/* the heart of the round trip: items become a numbered feedback block */
function feedbackText() {
  var title = document.title || location.pathname;
  var date  = new Date().toISOString().slice(0, 10);
  var lines = ['## Feedback on "' + title + '" (' + date + ')'];
  items.forEach(function (it, i) { /* …one line per annotation… */ });
  return lines.join('\n');
}

Shell sessions use the same pane with prompt and output tokens:

$ python3 -m http.server 8000
Serving HTTP on :: port 8000 (http://[::]:8000/) ...

The same ten classes cover any language. Elixir, for instance — atoms take the constant color, module attributes read as properties, and modules as types: 

defmodule Doc.Feedback do
  @moduledoc "Collects margin-note items the user pastes back."
  use GenServer

  def start_link(opts), do: GenServer.start_link(__MODULE__, [], opts)

  @impl true
  def handle_call({:add, item}, _from, items) do
    # newest first, like the feedback block itself
    {:reply, :ok, [item | items]}
  end
end

Diagrams

Diagrams are authored as Mermaid source and rendered client-side, themed with the same Mocha tokens. Mermaid's renderer is 3.4 MB, so it is not part of this template: the generating agent fetches mermaid.min.js once at authoring time and inlines it — but only into documents that actually contain a diagram. The document stays fully offline either way; when the renderer is absent, the source is shown as readable text in the same pane, which is exactly what you see below unless this copy was built with diagrams enabled:

Color belongs in the diagram source, not just the theme: one classDef per Catppuccin accent gives nodes the same colored-border-on-dark chrome your terminal UIs use, assigned per node — so the colors can mean something (here: green for the contract, mauve for the agent, blue for artifacts, peach for what you produce):

The annotation layer, retinted

margin-notes.js carries neutral built-in styling so it works dropped into any page; this template retunes it through CSS custom properties — --mn-box-bg, --mn-accent-bg, and friends — declared in :root. Comment boxes sit on mantle, and the copy button becomes a mauve pill, the same weight your terminal gives to anything titled. Try it: the button in the corner is live.
